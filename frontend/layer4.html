How is this calculated?

  • Live GitHub Analysis (50%): Fetches up to 30 public repositories to extract hard evidence of languages written (by byte count) and tags used, mapping them directly against JD required skills.
  • Code Quality Metrics (50%): Evaluates peer-validated quality indicators including Stars (max 30 pts), Forks (max 25 pts), documentation discipline, and commits made within the last 365 days.
  • Anti-Inflation Engine: Actively prevents candidates from padding their score with impressive-sounding project text without the underlying repository data to prove they built it.
  • Semantic Fallback: Gracefully degrades to a resilient neural TF-IDF/Cosine text comparison if a GitHub profile link is absent or if API limits are reached, ensuring a score is always generated.
